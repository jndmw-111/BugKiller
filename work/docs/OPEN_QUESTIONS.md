# 需要主办方确认的问题

1. 平台是否保证 `code/` 包含依赖缓存或允许本地离线安装依赖？
2. 平台对单条命令、总运行时间、测试次数和产物大小是否有统一上限？
3. 对需要数据库、中间件或容器的存量系统，平台提供哪些本地服务？
4. 最终提交包是否要求保留开发样例运行产生的 Trace 和证据？
5. 平台 Agent 是否统一支持命令超时参数，还是需要作品自行提供包装器？
6. 平台镜像预装哪些语言原生覆盖与变异工具，是否允许使用项目已锁定的
   本地工具依赖？

当前实现采用保守默认值：只读 `code/`、不联网下载脚本、限制扫描文件数
和读取字节数、所有测试命令要求显式超时。
