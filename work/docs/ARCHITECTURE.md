# 架构说明

本作品不包含额外的大模型 API 客户端。比赛平台提供的任意通用 AI Agent
都是智能决策核心，只要它能够读取 Markdown 和文件、运行本地命令并写入
产物：

```text
平台通用 AI Agent
  -> INSTRUCTION.md
  -> work/skill/SKILL.md
  -> 只读分析 code/
  -> 单次运行采用多种策略
  -> 完整启动或分层降级验证
  -> 动态生成并执行测试
  -> result/ + logs/
```

`openai.yaml` 是 Skill 生态要求的界面元数据，不表示作品依赖某个模型或
服务。核心指令均为普通 Markdown，辅助工具均为 Python 标准库脚本。

## 单次运行模型

五次评分运行互不累计。每次运行：

1. 生成独立 `run_id`。
2. 清理上一轮生成产物。
3. 从当前 `code/` 重新发现项目。
4. 至少使用三种相关漏洞策略。
5. 独立生成、执行、复测和报告。

## 复杂项目与降级

大型项目先按清单、子项目、入口和信任边界分层。完整系统启动失败时，
依次尝试现有测试、子项目、公共调用面、隔离假实现和纯逻辑测试。一个
模块失败不会阻止其他模块或策略继续。

## 数据边界

- 输入：`code/`
- 作品规则和工具：`work/`
- 生成产物：`result/`
- 审计记录：`logs/`

测试不得写回 `code/`。运行前后使用内容快照验证源文件未变化。
